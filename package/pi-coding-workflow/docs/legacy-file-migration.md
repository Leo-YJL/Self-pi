# Legacy File Migration Decisions

Status: package v0.1 slice.

The package is a TypeScript Pi package, not a copy of the old project-local workflow runtime. Legacy files are behavior references only.

## Decision categories

| Legacy category | Decision | Rationale |
|---|---|---|
| Old Python engine / `workflow.py` wrappers | Do not migrate | Package runtime is TypeScript. Reintroducing wrappers expands surface and compatibility burden. |
| Multiple old LLM-visible commands/tools | Do not migrate as tools | LLM-visible surface stays `workflow_next` / `workflow_run`. Stage actions are `workflow_run.action` values. |
| Old long prompts | Selectively distill | Rules should become skills, engine gates, templates or specs; avoid copying long prompt noise. |
| Project instance tasks/logs | Do not package | Use only as optional replay samples. They are project data, not package defaults. |
| GameBase private architecture rules | Do not hard-code | Generic package must work for non-GameBase Unity projects. Use Unity facts and optional overlays. |
| Mature runtime behavior | Migrate by capability | Map behavior into TS modules and tests, not by file name. |
| Historical telemetry/check/implement logs | Read as samples only | Current package uses schema-versioned runtime artifacts under `.workflow/.runtime/**`. |

## Current replacements

| Legacy behavior | Current package replacement |
|---|---|
| Route next workflow phase | `workflow_next` + `src/engine/route.ts` |
| Create/start/checkpoint/finish actions | `workflow_run` + `src/engine/run.ts` |
| PRD quality gates | `src/engine/prd.ts`, `src/engine/validate.ts` |
| Human final confirmation | `/workflow-prd-confirm` + `src/engine/prdConfirm.ts` |
| Implement/check file manifests | `src/engine/manifest.ts` |
| Checkpoint artifacts | `src/engine/checkpoint.ts` |
| Context budget / progressive disclosure | `src/engine/contextBudget.ts`, `src/engine/cache.ts` |
| Session state | `pi.appendEntry("pi-coding-workflow", ...)` in `src/index.ts` |
| Compaction state retention | `session_before_compact` hook + `src/engine/compaction.ts` |
| Base init | `/workflow-init` + `src/init/initWorkspace.ts` |
| Spec init | `/workflow-init-spec` + `src/init/specPlan.ts` |
| Unity facts | `src/init/unityScanner.ts` + Unity templates |

## Project output boundary

Package initialization may create or update:

```text
.workflow/config.json
.workflow/tasks/**
.workflow/spec/**
.workflow/.runtime/**
.gitignore
```

Package initialization must not create:

```text
.pi/extensions/**
old workflow Python wrappers
project-specific GameBase overlays unless explicitly generated by a future optional overlay feature
```

## Future migration candidates

- Historical task replay fixtures, stored under `tests/fixtures/` only after sanitization.
- Optional overlay mechanism for project-specific specs.
- Explicit rollback action after transaction rollback policy is designed.
- Git finalizer after protected branch / staging / push confirmation policy is finalized.
